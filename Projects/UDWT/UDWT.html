Introduction

This project contains specific code for comparing electrostatic charge map implementations.

Build Options

Note that not all of these options have been fully tested in all combinations. If you find a problem combinations please inform Alex Cole (t-alcole). Also note that you will need to rebuild "Common" with the same parameters to use these properly.

  • NO_ACCELERATOR - Exclude the Accelerator target from the build. Note that this project uses Float4 so requires at least one Acclerator target.
  • NO_ACCELERATOR_OPT - Exclude the optimised Accelerator target from the build. Note that this project uses Float4 so requires at least one Acclerator target.
  • NO_CUDA - Exclude the CUDA target from the build.
  • NO_CUDA_OPT - Exclude the optimised CUDA target from the build. Note that this is not supported at all in the Molecules project as there is no optimised CUDA code.
  • NO_OPEN_CL - Exclude the OpenCL target from the build.
  • NO_REFERENCE - Exclude the Reference (CPU) target from the build.
  • NO_REFERENCE_OPT - Exclude the optimised Reference (CPU) targets (most have two versions) from the build.
  • NO_STATIC_DATA - When generating arrays of input data, a number of methods can be used, including random, 0s and pre-set data. Using the pre-set data may make the executable huge, so can be excluded using this switch.

Launch Options

This program takes five options, all passed in order.

  • X grid size index (0 - 30). Determines the x size of the grid to operate on, based on the values in the array in Main.cpp
  • Y grid size index (0 - 30). Determines the y size of the grid to operate on, based on the values in the array in Main.cpp
  • Atom count. Number of atoms to place within the grid.
  • Targets. Which targets to run the test on. Options are (not all may be supported):
    • a - DX9
    • b - DX9 Optimised
    • c - CUDA
    • d - CUDA Optimised
    • e - Cached (Reference Optimised)
    • f - Fast (Reference Optimised 2)
    • r - Reference
    • o - OpenCL
    • x - X64
    • y - X64 Optimised
  • Experiments. Which experiments to run. Options are (note that using "S" over "C" is recommended):
    • s - Setup. Run all selected targets multiple times and time the results. This mode includes all data copying in the results.
    • c - Clean. Run all selected targets multiple times and time the results. This mode does not include data copying where possible in the results.
    • v - Verify. Run all selected targets once and compare their results.
